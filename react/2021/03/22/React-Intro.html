React 소개

by kt1201

March 22, 2021
in React

프론트엔드 라이브러리/프레임워크

프로젝트가 사용자와의 상호작용이 별로 없다면, 프론트엔드 라이브러리는 필요하지 않다. 하지만 프로젝트가 규모가 커지고, 다양한 유저 인터페이스와 상호작용을 제공하게 된다면, 많은 DOM 요소들을 직접 관리하고 그 코드를 정리하는건 정말 힘든 일이다. 웹 개발을 하게 될 때, 귀찮은 DOM 관리와 상태값 업데이트 관리를 최소화하고, 오직 기능 개발과 UI를 구현하는 것에 집중 할 수 있도록 하기 위해서 프론트엔드 라이브러리와 프레임와크들이 만들어 졌다.

그 중에서 페이스북에서 만든 React에 대해서 소개할 것이다.

Virtual DOM

React 이전의 프레임워크들은 데이터단의 Model, 사용자의 화면에서 보여지는 View, 사용자가 발생시키는 이벤트를 처리해주는 Controller로 이뤄진 MVC패턴, 여기서 파생된 MVVN, MVW 등의 패턴으로 이뤄져 있다.

여기서 공통점은 Model이다. 양방향 바인딩을 통하여 Model에 있는 값이 변하면, View에서도 이를 변화시켜준다. 페이스북은 이 변화라는 것을 하지 말자, 데이터가 바뀌면 그냥 뷰를 날려버리고 새로 만들자 라는 생각을 했다. 이때 Model의 데이터가 바뀔 때마다 View를 새로 생성하면 성능적으로 큰 부담이 될 것이다. 그래서 나온것이 Virtual DOM이다.

DOM에 변화가 생기면 렌더트리를 재생성하고(모든 요소들의 스타일이 다시 계산된다.) 레이아웃을 만들고 페인팅을 하는 과적이 다시 반복된다. 변화를 적용하기 위해 브라우저가 많이 연산을 해야한다는 소리고, 전체적인 프로세스를 비효율적으로 만든다.

이 부분에서 Virtual DOM이 빛을 발한다. 만약 View에 변화가 있다면, 그 변화는 실제 DOM에 적용되기 전에 Virtual DOM에 먼저 적용시키고, 그 최종적인 결과를 DOM으로 전달해준다. 이로써, 브라우저 내에서 발생하는 연산의 양을 줄이면서 성능이 개선되는 것이다.

다시 쉽게 말하면, Virtual DOM을 사용함으로써, 데이터가 바뀌었을 때 더이상 어떻게 업데이트 할지를 고려하는게 아니라, 일단 바뀐 데이터로 그려놓고, 비교를 해서 바뀐 부분만 찾아 바꿔준다. 그러므로 DOM fragment를 관리하는 과정을 수동을 하나하나 작업할 필요 없이, 자동화하고 추상화 하는 것이다.

React를 명료하게 표현한다면, ““최적화 작업을 자동화해주는 React를 사용했을 때 생산성이 배가 된다.” 이다.

React의 장점

React는 정말 자유도가 높은 라이브러리이다. 예로, React 컴포넌트 스타일링을 할 때도 한가지 정해진 방식이 있는게 아니라 수많은 방식으로 할 수 있다.

React 라이브러리는 View쪽만 관리하게 하고, 나머지 기능은 써드파티 라이브러리가 담당하게 함으로써 개발자들의 다양한 시도와 연구를 통하여 계속해서 성장해 나아갈 수 있다.
