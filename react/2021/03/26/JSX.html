코드이해

전에 만들었던 리액트 프로젝트의 src/App.js 를 보자

/* src/App.js */

import React from 'react';
import logo from './logo.svg';
import './App.css';

function App() {
  return (
    <div className="App">
      <header className="App-header">
        <img src={logo} className="App-logo" alt="logo" />
        <p>
          Edit <code>src/App.js</code> and save to reload.
        </p>
        <a
          className="App-link"
          href="https://reactjs.org"
          target="_blank"
          rel="noopener noreferrer"
        >
          Learn React
        </a>
      </header>
    </div>
  );
}

export default App;

위에서부터 한줄씩 보자

리액트 프로젝트를 생성하면 node-modules라는 디렉터리가 생성되는데 이 안에 react 모듈이 설치된다. 아래 명령어를 통하여 리액트를 불러와 서용할 수 있다.

import React from 'react';

이렇게 모듈을 불러와서 사용하는 방법은 사실 원래 브라우저에 없던 기능이다. 이러한 기능을 브라우저에서도 사용하기 위해 bundler를 사용한다. 파일을 묶어서 하나의 파일로 연결하는 것이다.

대표적인 bundler로 웹팩을 많이 사용하는 추세이다. 웹팩을 사용하여 SVG와 CSS 파일도 불러와 사용하는 코드는 아래와 같다.

import logo from './logo.svg';
import './App.css';

그 아래 function App()으로 App이라는 컴포넌트가 하나 생성되어 있다. function 키워드를 통하여 만들어졌으므로 함수형 컴포넌트라고 부른다. 이 App 컴포넌트에 있는 코드의 형식을 JSX라고 부른다.

JSX란?

App 컴포넌트식으로 작성한 코드는 브라우저에서 실행되기 전에 코드가 번들링 되는 과정에서 바벨이라는 것을 사용하여 일반 자바스크립트 형태의 코드로 변환된다.

funciton App() {
    return (
        <div>
            Hello <b>react</b>
        </div>
    );
}

위와 같은 코드는 아래와 같이 변환된다.

function App() {
    return React.createElement("div", null, "Hello ", React.createElement("b", null, "react"));
}

위 두 코드를 보면 JSX가 훨씬 가독성도 높고 작성하기도 쉽다고 느껴진다. 사실상 이것이 JSX를 사용하는 주된 이유이다.

또한 src/index.js파일을 열어보면 이 App 컴포넌트를 import하여 마치HTML 태그 쓰듯이 그냥 작성하는 코드를 볼수 있다. 이와 같이 높은 활용도를 보여준다.

JSX 문법

JSX를 올바르게 사용하기 위해서는 몇가지 규칙이 있다.

감싸인 요소

컴포넌트에 여러 요소가 있다면 반드시 부모 요소 하나로 감싸야 한다. App.js를 지우고 아래와 같이 작성해보자

/* src/App.js */

import React from 'react';

function App() {
  return (
    <h1>리액트 안녕!</h1>
    <h2>잘 작동하니?</h2>
  );
}
export default App;

터미널과 웹페이지에 에러가 날 것이다. 아래와 같이 해결할 수 있다.

/* src/App.js */

import React from 'react';

function App() {
  return (
    <div>
      <h1>리액트 안녕!</h1>
      <h2>잘 작동하니?</h2>
    </div>
  );
}
export default App;

리액트 컴포넌트에서 요소 여러 개를 하나의 요소로 감싸주어야 하는 이유는 Virtual DOM에서 컴포넌트의 변화를 감지해 낼 때 효율적으로 비교할 수 있도록 컴포넌트 내부는 하나의 DOM 트리 구조로 이루어져야 한다는 규칙이 있기 때문이다.

<div> 말고 리액트 v16부터는 <Fragment>도 사용 가능하다. <Fragment>는 <>...</>와 같이 쓸 수도 있다.

#####

자바스크립트 표현

JSX안에서는 자바스크립트 표현식을 쓸 수 있다. 코드는 아래와 같다.

/* src/App.js */

import React from 'react';

function App() {
  const name = '리액트';
  return (
    <div>
      <h1>{name} 안녕!</h1>
      <h2>잘 작동하니?</h2>
    </div>
  );
}
export default App;

const는 한번 지정하면 변경이 불가능한 상수를 선언한때 사용

let은 동적인 값을 담을 수 있는 변수를 선언할 때 사용

const와 let은 같은 블록 내부에서 중복 선언이 불가하다.

let a = 1;
let a = 2; // 오류

if문 대신 조건부 연산자

JSX 내부의 자바스크립트 표현식에서 if문을 사용할 수 없다. JSX 밖에서 if문을 사용하여 사전에 값을 설정하거나, {} 안에 조건부 연산자를 사용하면 된다.

/* src/App.js */

import React from 'react';

function App() {
  const name = '리트';
  return (
    <div>
      {name === '리액트' ? (
        <h1>리액트입니다.</h1>
      ) : (
        // <h2>리액트가 아닙니다.</h2>
        null
      )}
    </div>
  );
}
export default App;

AND 연산자(&&)를 사용한 조건부 렌더링

위 코드를 AND 연산자를 이용하여 아래와 같이 더 짧게 바꿀 수도 있다.

/* src/App.js */

import React from 'react';

function App() {
  const name = '리트';
  return (
    <div>
      {name === '리액트' &&
        <h1>리액트입니다.</h1>
      }
    </div>
  );
}
export default App;

예외적으로 false의 값이 0이면 화면에 0이라고 나타난다.

const number = 0;
return number && <div>내용</div>

인라인 스타일링

리액트에서 DOM요소에 스타일을 적용할 때는 문자열 형태로 넣는 것이 아니라 객체 형태로 넣어 주어야 한다. 예로 background-color는 backgroundColor처럼 ‘-‘문자열을 없애고 카멜 표기법(camelCase)으로 작성해야 한다.

/* src/App.js */

import React from 'react';

function App() {
  const name = '리액트';
  const style = {
    backgoundColor: 'black',  // background-color
    color: 'aqua',
    fontSize: '48px',  // font-size
    fontWeight: 'bold',  // font-weight
    padding: 16  // 기본 단위는 px
  };
  return <div style={style}>{name} </div>
}
export default App;

class 대신 className

일반 HTML에서 CSS 클래스를 사용할 때는 <div class="myclass"></div>와 같이 사용한다. 하지만 JSX 에서는 class가 아닌 className으로 설정해 주어야 한다.

App.css를 지우고 새로 작성해주자.

/* src/App.css */

.react {
  background: aqua;
  color: black;
  font-size: 48px;
  font-weight: bold;
  padding: 16px;
}

react 클래스를 사용하는 코드는 아래와 같다.

/* src/App.js */

import React from 'react';
import './App.css';

function App() {
  const name = '리액트';
  return <div className="react">{name} </div>
}
export default App;

꼭 닫아야 하는 코드

HTML 코드를 작성 할때 가끔 태그를 닫지 않은 상태로 코드를 작성하기도 한다.

하지만 JSX에서는 태그를 닫지 않으면 에러가 발생한다. self-closing 태그도 가능하니 꼭 닫아주도록 하자.

주석

JSX내부에서 주석은 일반 javascript 주석과는 다르다.

// or /*...*/와 같은 주석은 페이지에 그대로 출력된다.

JSX 내부에서 주석은 {/*...*/}와 같이 작성하자.

클래스형 컴포넌트

컴포넌트를 선언하는 방식은 두가지이다. 하나는 function을 이용하여 선언하는 함수형 컴포넌트이고, 또 다른 하나는 클래스형 컴포넌트이다.

클래스형 컴포넌트는 아래와 같다.

/* src/App.js */

import React, { Component } from 'react';
import './App.css';

class App extends Component {
  render() {
    const name = 'react';
    return <div className="react">{name}</div>
  }
}
export default App;

역할은 함수형 컴포넌트와 같다.

차이점은 클래스형 컴포넌트의 경우, 임의 메서드를 정의할 수 있다는 것이다.

클래스형 컴포넌트에서는 render() 함수가 반드시 있어야 하고, 그 안에서 보여 주어야 할 JSX를 반환해야 합니다.

그렇다면 컴포넌트를 선언할 수 있는 두가지 방법 중 어느 상황에 함수형 컴포넌트를 사용해야 할까? 함수형 컴포넌의 장점은 아래와 같다.

  • 클래스형 컴포넌트보다 선언하기 편함
  • 메모리 자원 덜 사용
  • 결과물의 파일 크기가 더 작음(사실상 별 차이는 없음)

기존의 단점인 state와 라이프사이클 API의 사용 불가는 리액트 v16.8 업데이트 이후 Hooks라는 기능이 도입되면서 해결되었다.

리액트 공식 매뉴얼에서는 컴포넌트를 새로 작성할 때 함수형 컴포넌트와 Hooks를 권장하고 있다. 하지만 클래스형 컴포넌트가 사라지는 것은 아니므로 클래스형 컴포넌트의 기능도 꼭 알아두어야 한다.

Hooks를 익히기 전까지는 클래스형 컴포넌트 위주로 사용하여 리액트의 기본기를 익히자.
