Container vs Virtual Machine

소프트웨어는 OS와 라이브러리에 의존성을 띈다. 그러므로 하나의 컴퓨터에서 성격이 다른 소프트웨어를 한번에 실행할 때 어려움을 가질 수 있고 관련된 구성을 관리하기가 어렵다. Container는 개별 소프트웨어의 실행에 필요한 실행환경을 독립적으로 운용할 수 있도록 기반환경 또는 다른 실행환경과의 간섭을 막고 실행의 독립성을 확보해주는 운영체계 수준의 격리 기술을 말한다. Container는 애플리케이션을 실제 구동 환경으로부터 추상화할 수 있는 논리 패키징 메커니즘을 제공한다.

구조는 아래 그림과 같다. Container, VM 순이다.

VM 또한 독립적인 실행환경을 구성할 수 있도록 도와주는데 차이는 다음과 같다.

  • VM

    HyperVisor는 컴퓨터가 가지고 있는 인프라 리소스들에 대해 VM별로 배분하는 역할을 한다. 또한 각 VM에서는 독립적인 Guest OS를 가지고 있다.

    따라서 독립적인 플랫폼을 하나씩 증가시킬 때마다 불필요한 OS를 만드는 작업에 대해서 계속해서 해야한다. 즉 확장성이 떨어진다. 또한 메모리나 자원에 관해서 유동적으로 관리되는게 아니라 처음부터 정해놓고 실행하기에 비효율적이다.

  • Container

    컨테이너의 경우 하나의 Host OS위에서 마치 각각의 독립적인 프로그램처럼 관리되고 실행된다. 불필요한 OS 만드는 작업 및 Infra를 독립적으로 나눌 필요가 없어서 확장성이 좋고 빠르다.
